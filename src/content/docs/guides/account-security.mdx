# Account Security

Manage your account security, active sessions, and privacy settings to keep your account safe.

## Accessing Security Settings

1. Click your avatar in the top-right corner
2. Select **Settings**
3. Select the **Security** tab
4. Or navigate to `trucklinemp.com/user/me/settings?tab=security`

---

## Connected Account (Steam)

Your Steam account is permanently linked to your TrucklineMP account.

- **Status:** Cannot be disconnected
- **Purpose:** Primary identity verification
- **Steam ID:** Displayed in this section

This connection is required for account verification and cannot be changed.

## Two-Factor Authentication (2FA)

Add an extra layer of security to your account.

### Enabling 2FA

1. Click **Enable 2FA**
2. A QR code is generated
3. Scan the QR code with your authenticator app:
   - Google Authenticator
   - Authy
   - Microsoft Authenticator
   - Any TOTP-compatible app
4. Or manually enter the secret key shown below the QR code
5. Click **Continue to Verification**
6. Enter the 6-digit code from your app
7. Click **Verify & Enable**
8. **Save your backup codes** — they won't be shown again

### Backup Codes

When you enable 2FA, you receive **10 backup codes**:
- Each code can be used **once**
- Use them if you lose access to your authenticator app
- Store them in a safe place (password manager, printed copy)
- You can copy all codes at once

### Disabling 2FA

1. Click **Disable 2FA**
2. Enter your authenticator code **or** a backup code
3. Click **Disable 2FA**
4. 2FA is now removed from your account

**Warning:** Disabling 2FA makes your account less secure.

### If You Lose Your Authenticator

1. Use a backup code to sign in
2. Once signed in, go to Security Settings
3. Disable 2FA using the backup code
4. Re-enable 2FA with a new authenticator app

**Without backup codes:** Contact support via Discord with proof of identity.

## Developer Mode

Enable the Developer Console to access professional tools.

### What You Get
- Create API projects
- Manage API credentials
- Access developer documentation
- Build integrations with TrucklineMP

### Enabling Developer Mode
1. Click **Activate** in the Developer Mode section
2. Read the confirmation dialog
3. Click **Confirm Activation**
4. Developer mode is now active

**Warning:** This action is permanent and irreversible.

### Accessing the Console
- Once enabled, click **Open Console** to access developer tools
- Or navigate to `trucklinemp.com/developer`

## Active Sessions

View and manage all devices currently signed into your account.

### Session Information
Each session shows:
- **Browser** — Chrome, Firefox, Safari, Edge
- **Operating System** — Windows, macOS, Linux, Android, iOS
- **IP Address** — The IP the session originated from
- **Start Date** — When the session was created
- **Current badge** — Marks your current session

### Revoking a Session
1. Find the session you want to revoke
2. Click **Revoke** next to it
3. The session is immediately signed out
4. The device will need to sign in again

### Signing Out All Other Sessions
1. Click **Sign out all other sessions** at the top
2. All sessions except your current one are revoked
3. Useful if you suspect unauthorized access

## Security Best Practices

### Essential
- **Enable 2FA** — The single most important security step
- **Save backup codes** — Store them in a password manager
- **Review sessions regularly** — Revoke anything you don't recognize
- **Use a strong password** — Unique to TrucklineMP

### Recommended
- Sign out on shared devices
- Review authorized apps periodically
- Keep your connected accounts up to date
- Report suspicious activity immediately

## Troubleshooting

### "I can't enable 2FA"
- Ensure your authenticator app supports TOTP
- Check that your device's time is synchronized
- Try a different authenticator app

### "I lost my backup codes"
- If still signed in, disable 2FA and re-enable it
- If locked out, contact support via Discord
- Provide proof of identity for account recovery

### "I see a session I don't recognize"
- Revoke the session immediately
- Change your password if possible
- Enable 2FA if not already enabled
- Contact support if you suspect unauthorized access

### "I can't sign in with 2FA"
- Check that your authenticator app's time is correct
- Try a backup code
- If all else fails, contact support
